Dennis,

If you're talking about framebuffered terminals, then these are a few 
basic things to check first:

  1) that the kernel was compiled with these enabled:

     bash$ cd /usr/src/linux
     bash$ make menuconfig

	{ Device Drivers }>
           { Graphics Support }>
	      [*] VESA VGA graphics support
	      { Console display driver support }>
                 [*] Video mode selection support
	         <*>Framebuffer Console supports

  2) that your bootloader has the appropriate "video=" line.  I use grub 
     to load my terminal 1280x1024 with something like this:

        root (hd0,0)
	kernel /kernel-2.6.11 root=/dev/hda6 video=vesafb:ywrap,mtrr vga=0x31A

     I grabed that off of gentoo's forums, which has about three good 
     threads on the topic.  If you'd like to adjust the resolution to
     fit your laptop, you'll change the "vga=" part to what's listed in 
     the table shown on the gentoo wiki:

	http://gentoo-wiki.com/HOWTO_Framebuffer:Bootsplash:Grubsplash

BTW, What type of video card is it?  Could you post up an lspci next 
time you're around the laptop?  It could be the case that it doesn't support 
the VESA standards, but I doubt it.
