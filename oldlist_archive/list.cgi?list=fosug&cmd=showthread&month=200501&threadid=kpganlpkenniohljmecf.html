Hey guys... I need some advice.

I'm about to embark on my least favorite of journeys again... Mail servers.

Here's what's up:

At work, we run a mostly windows network.  The PDC and DHCP servers are win2k
servers, the web (Internet Infection Sharing (IIS)) mail (iMail) server is NT4
(yuck...).  We have an AIX server that serves as a product database (about
350,000 sku's.)

I'm working on bring up our Intranet web site.  On the site, I'm putting in a
help desk ticket system (OSTicket).  The server will be running Linux, probably
Debian.  It'll have Apache, MySQL, PHP, Perl - the whole gambit for a web
server.

I'm trying to decide on the MTA and I'm a little reluctant.  I've never had
_any_ luck at all getting my MTAs to work for me... Postfix, Qmail, Sendmail -
whatever.

What is supposed to happen is when a user submits a ticket with OSTicket, he
receives an email with the ticket number and the IT staff receives a
notification that a ticket has been submitted.  There does not have to be two
way communication since the emails are only notifications and I want to keep
the end user using the web interface to add to their trouble ticket.

I just need the Linux server to send the outgoing email to the mail server so it
gets to the EU.  Is there anything special that I'll need for this kind of
configuration?  The server will not be accessed from outside the network, and
should not send email to anywhere other then the company domain.

What would you guys recommend?
