Not sure if anyone has played with the (GASP) Gui managers for the
portage.  There have been a few programs out there to make a gui for
the gentoo portage. xportage, portage-master, kportage, gportage, etc.
Saying they were a letdown is a understatment in my opinion Poor
searching  and management capabilites.  Horribly slow indexing of
portage. Inability to use advanced command line switches. Etc.  I Came
accross a new one. Which although is not perfect in my opinion has
made major leaps in a good GUI portage manager.  This program is
called "porthole". IF you have not tried it out yet, I suggest giving
it a try.   Very nice even if you just use it to search and view the
portage tree. It supports a large variaty of options, It allows you to
change use flags and march type per portage entry.  And another cool
feature is, after it's learned your system it will be able to estimate
the time it will take to emerge certain things.

Another great portage utility is "esearch"  If you run many search's
"emerge -s porthole" etc you will find that these searches can take
minutes or even longer, Especialy if you use the -S (search
description) options.  esearch will index your portage and return your
searches in seconds.  "esearch -s porthole". I have a weekly cron job
that grabs the newest portage, after the portage is done I have eseach
update it's database. This usually takes 10-15 minutes, but i'm asleep
so who cares. It greatly speeds up searching while I'm awake.
