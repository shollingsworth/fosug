I am sure Robert will chime in on this one as I think I heard him mention
the software before but I have to put my two cents in here as well.  I own a
linksys wrt54g v2.0 and just flashed this with DD-WRT v23.   I don't know if
any other source code projects are this great or not but I absolutely love
this firmware.  I CAN USE VLAN's ON THIS ROUTER!!!!  That just blows my
mind.  I wish some of the help pages were more up to date in the router as
the manual on the site is in german, but other than that it amazes me what
this thing can do.  Oh yeah.  Before I forget, QoS was the other big one.
You can prioritize traffic.  Need to make sure p2p apps don't suck up your
bandwidth?  This thing does it.  Want to run a hotspot?  This has a
configuration page that will let you set up stuff on it.

I am really curious to see what other versions are out there.  Sveasoft is
supposed to be another one that went commercial with their project and are
charging for the software via a membership or something.  I just stuck with
the dd-wrt fork.  I didn't read too much about the whole situation as I was
just interested in the software.

Right now I am lucky enough to own two of these routers (a friend replaced
his with another brand for some reason) and I flashed the backup with the
DD-WRT and am trying it live for a while.

Here are my thoughts on both distros:

DD-WRT

Pros:

--Easy to install
--Micro edition will install into a v5 WRT54G (though as I'd said, it 
takes a bit of doing)
--Very complete, even if you look at the Micro's feature set
--Seems to be very cutting-edge (though I believe it is indeed much of 
Sveasoft's doing)
--Nearly all features are web-configurable
--Can use OpenWRT's packages
--Well-documented and -supported on forums and wiki

Cons:
--If you want functionality which require additional kernel modules, you 
can't use OpenWRT's packages; you'll have to build your own kernel, 
which isn't necessarily a bad thing.
--Web interface seems a bit persnickety at times.
--SSL VPN is still too young, especially on the Windows desktop.  I 
ended up going with straight-up PPTP, which I realize ain't the most 
secure thing in the world, but right now, it's better than nothing.

OpenWRT

Pros:

--Very actively developed.
--Package system
--Well-documented and -supported on forum and wiki
--Very stable and usable

Cons:

--Web configurability is sparse at best, even on standard features.
--Seems a little behind the curve of some of the more advanced features, 
including VPN functionality
--No ability to go into a v5 WR54G (though I wouldn't touch one of those 
to begin with unless I had certain reasons to justify it and I had $50 
to guiltlessly blow)

Both

Pros:

--Telnet accessibility!
--SSH accessibility!
--VPN accessibility!
--The instant I connect to it with telnet, it's like putting a fish in 
water.  Hello, Linux command prompt~!
--Has the ability to, based on hardware, use additional space via other 
means, whether using USB media (flash cards, USB thumb drives, flash 
adapters or even HDDs)
--Brings enterprise-type functionality to the average user, like VLANs, 
routing protocols (e.g. OSPF, RIP, BGP), etc
--Lots of cool apps we all know and love have been ported, even Asterisk!
--You could even replace some of your "bigger iron personal datacenter" 
gear with these little bad boys  Imagine an authoritative DNS server 
which rarely has to deal with moving parts!

Cons (mostly because of the devices the firmware runs on):

--Never enough RAM to run everything you wanted
--Never enough flash to store a whole lot of you want to keep on it if 
the unit travels
--IPsec may take some doing, or gear specifically designed to handle it 
(like one NetGear unit). Some ciphers or lots of tunnels require a bit 
of processing power which this processor may not be up to the task of 
handling.
--No FPU for some stuff which requires hardware FP, like iLBC and G.729 
for Asterisk.

> I don't know if multiple SSID is really all that great a bag.  If you go with
> openwrt, it has full iptables and you can do mac based filtering. [2]

I s'pose there's that.  Though I'd like it to be as transparent to me as 
possible.  Just wanna give my roomie a SSID, if applicable, a WPA 
passphrase or setup for 802.1x, have her boot up and go!  Saves me a lot 
of administrative work (and another outlet so I don't need to plug in 
yet another brick).

--Ian.
