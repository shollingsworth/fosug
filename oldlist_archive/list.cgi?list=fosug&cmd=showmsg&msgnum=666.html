Anyway, for those who think that 012345678901234567890123456 or
123456789012345678901234567 are ok for a secure WEP key [0], I threw together
this perl code to help.  It will generate a random [1] 26 character (128-bit)
WEP key for your use.

-------------------- Begin --------------------
#!/usr/bin/perl -w
#
# Generate a random 128-bit WEP key.
#

use strict;

my @FNchars = ();
# If you need to generate a different length key, just change keylen!
my $keylen = 26;

my $key = "";

for(48..57,65..90,97..122) { push @FNchars, chr($_); }

srand (time ^ $$ ^ unpack "%L*", `ps axww | gzip`); #generate some randomness.

for(0..$keylen) { $key .= $FNchars[rand($#FNchars)]; }

print "Your key sir: $key \n";

-------------------- End --------------------
