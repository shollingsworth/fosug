I'm seeing quite a bit of them on some of the sites I see on my own
server.  It looks like mod_security's getting quite the workout.

I still use Apache 1.3.x presently because I've yet to be convinced that
Apache 2.x is stable enough for production use (quite a few security
issues).  I've heard, though, that with mod_fastcgi and Apache 2.x, if
you use PHP, you'll see a noticeable decrease in memory utilization,
especially on busy webservers hosting lots of sites.  This also allows
Apache 2.x to run in threaded mode (PHP as an Apache module will not).
