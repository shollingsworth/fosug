Hi everyone,

Power outage season is here. The first outage just happened to us this  
morning (in the Alluvial/Willow area of Clovis), and well, it's always  
a little iffy when a running computer suddenly loses power.

I'm no sysadmin, and no expert on file systems either, so I'm not very  
familiar about how robust the ext3 (for instance) file system is in  
these cases. Is the data recoverable in "almost all" cases? As for  
MySQL, I think tables can become corrupted if the system loses power  
in the middle of a transaction. A "repair table" seems to fix it but  
it's an interruption.

Yes I know, backups are always a good idea.

We're running a couple of desktop systems in our office/living room.  
No power-hungry servers, and nothing that can't be "off" for the  
duration of an outage.

So, I started looking for cheap, low-budget UPS packages. Any  
recommendations? Anything worthwhile in the $25-$50 range? All we  
really need is something that can maintain power long enough to shut  
down the machines in an orderly fashion, although it would be really  
nice if the UPS would cause the machines to shut down automatically  
rather than having to do it manually once the UPS starts beeping...

Is UPS yet another area where I need to check for Linux hardware  
support before buying?

Many questions. TIA for any pointers.

Mattias

Mattias,

I've had pretty good luck with the low-end APC models. I have a BackUPS-ES
750 and a BackUPS RS 800 supporting a few machines at my apartment. Each is
supported by the free apcupsd (www.apcupsd.com) (OSS, works on FreeBSD, I
can only assume it also works on Linux). I think the hardware cost me
something like $70 or so.

If you have old UPSes (doesn't really matter how old) laying about (working
or otherwise), APC has a cool TradeUPS program. You buy a new UPS from them
at a decent discount, and send the old unit back to them for safe disposal.
I'm pretty sure they pay for return shipping.

Cheers,
Sean

http://www.apc.com/tools/ups_selector/index.cfm

I've got APC BackUPS Pro 650s on all the machines at work. Server (P3-800 3x
10K SCSI, no monitor) is on a SmartUPS 1000.

I have a BackUPS Pro 420 or something at home that isn't enough for the Athlon
850 MHz space heater + 17" monitor. Or the battery is toast again. It pretty 
much shuts off immediately on power failure.

I have pretty much always gotten 3 years out of batteries, unless they are
used heavily and drained way down.

Oh there is one thing that will trash a UPS: removing the neutral on the line
side power. No, I did't do it. No, they weren't salvagable...I tried. LOL.

One guestimate. Add up the watts of all of your stuff that you want backed up.
Divide that number by .8. Use that number for the VA size. (For those wonder
WTF the .8 means, it's the power factor that UPSes are at.) You do lose quite
a bit of power to heat in the conversion from DC to AC.

As for montiring software, I've always used APCs software, Powerchute.

I'm using an APC 350 VA connected via USB to my desktop machine.  My
monitor and any peripherals are NOT connnected to battery backup, but
are surge protectes.  Once this was done, I installed the APCUPSD
package (in debian) which monitors signas from the UPS.  With a little
help from the readme I was able to setup the config file in a couple
of minutes.

As a result, this is what I get.
When power is first lost, the machine does nothing.  If power is not
restored within 60 seconds, (or the battery reaches <50% charge
remaining), it runs a script that ssh's to my file server (which is on
a dumb APC unit, along with the switch that connects the two machines)
and shuts it down, then proceeds to shut itself down.

I believe I saw the 350 at office depot on Shaw and Brawley for ~$30
on tuesday. (might be after rebate)

Also, compusa has the 800 on sale for like $80 after rebate
(http://dealnews.com/deals/APC-Back-UPS-800-VA-UPS-for-70-after-rebate/119805.html)
