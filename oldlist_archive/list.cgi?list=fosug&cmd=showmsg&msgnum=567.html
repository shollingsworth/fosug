Looking through RFC 2476:

What soemone at SBC had denoted as "port 587" is more commonly
referred to as ESMTP. 

That, presumably, will be supported with every single common MTA.


So, in revised summary: 


If a site does provide SMTP service, and must be connected-to, from 
anyone who would be connecting through SBC, then the site's
administrator will  probably want to support ESMTP connections, in
the site's SMTP server -- like, very soon -- or the people using SBC
will not be able to send emails, via the site's SMTPN server.


Furthermore, anyone using SBC will have to use  either:
1) SBC's MTA
2) an MTA supporting ESMTP
3) maybe some SSL tunneling ?

(n.b SSL tunneling is doable for us, but not be so doable, for
everyone.)


I'd say SBC has overdone it. I'll keep any comments to myself, about
their partner, Yahoo.
