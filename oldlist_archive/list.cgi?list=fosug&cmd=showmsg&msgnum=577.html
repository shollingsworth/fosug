Actually, I will put my two cents in.  I apologize for any line wrappage.

Here's my port 587 config for my own server:

submission       inet    n       -       n       -       10
smtpd
                 -o content_filter=
                 -o 
smtpd_recipient_restrictions=permit_mynetworks,permit_sasl_au
thenticated,reject
                 -o smtpd_helo_restrictions=
                 -o smtpd_client_restrictions=
                 -o smtpd_sender_restrictions=

Very simplistic since I know that I'll permit anything from my own IPs 
and IPs I completely trust carte blanche, allow anything which has an 
actual account on the server THAT HAS BEEN AUTHENTICATED FOR.  The rest 
are told to shove it where the sun don't shine.

I'm not all that concerned about ETRN, though I really should turn it 
off for port 587 per RFC 2476.
