If you're on KDE, then kuser is an admin tool much like MS User Administrator.
Close enough you can muddle through it.

There are really no set user levels like 2K or XP. As far as being able to
trash the system, either you're root or not.

Every file has ownership as USER(owner) GROUP EVERYONE. (There's also SUID and
GUID, but that's further down the road.) They can have read, write and
execute. I always screw them up but here's something close:

0 = No permissions
1 = execute
2 = write
3 = execute + write
4 = read
5 = execute + read
6 = write + execute
7 = execute + write + read

I think I may have gotten that right, not sure. And you also can't lock out
root's access to a file like you can lock the Administrator out of files on
Windows (BAD BAD BAD). This is the standard permissions. There are UNIX ACLs
as well, but most people don't use them. They give finer grained controls,
much like Windows.
