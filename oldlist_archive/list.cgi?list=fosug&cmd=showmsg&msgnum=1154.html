Hi All,
I came across a really good set of tools offered up by Yahoo! under a  
BSD license (which can be reissued under GPL i think). There is a  
full UI library that I have not tested yet (AJAX, DHTML) but the CSS  
grid portion looks really promising.
 From their site:

[snip]
Grids CSS is a suite of seven web page templates and the ability to  
nest grids of one to four columns within the content area of those  
templates. Together, the combined template and grid system offers  
these features:

     * Full A-grade browser support
     * More than 100 page layouts with a single CSS file at just 1.82KB
     * Flexible scaling with user font-size adjustments
     * Min-width enforcement, regardless of font or viewport size
       (except IE; helper JS under consideration)
     * Abundant DOM hooks for CSS and scripting flexibility
     * Source-order independence, allowing you to place your most  
important
       body content first in your HTML document
     * Self-clearing footer (no matter which column is longer, the  
footer stays at the bottom)
     * Centering within the viewport by default
     * Forward-compatibility as standard page structure evolves to  
claim more screen real estate
     * Accommodation for IAB Advertising Units
[/snip]

Yahoo! User Interface Library
http://com1.devnet.scd.yahoo.com/yui/

Grids CSS
http://com1.devnet.scd.yahoo.com/yui/grids/

I'm not enthralled with all of the code but it's pretty dang clean  
and could go a long way in helping with better standards compliance.  
Stylesheets are broken out in an interesting/logical way. In the CMS  
world this could provide the backbone for a whole new theming system.  
(lots of buzz on the drupal lists about the CSS grid).

Have fun!
suz
