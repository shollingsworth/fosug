On 2005.07.22 22:42:42 +0000, Mike Fennell wrote:
> Hi Guys,
> 
> Got a question that I hope someone that can help me out with..
> 
> I need to allow remote access to a mysql database. Sounds simple, I've done
> this before, just not on this particular machine. I have done the following:
> 
> GRANT ALL PRIVILEGES ON *.* TO USERNAME@IP IDENTIFIED BY "PASSWORD";
> 
> That goes through just fine then I flush the privileges and all should be
> well at that point, but the user is getting a access denied error. So I
> checked the firewall, ports were open, so I disabled the firewall (iptables
> with the APF script) and still nadda.. no mysql love.
> 
> I have quadruple check that I used the correct username and password the
> client swears he's using the correct info. 
> 
> Changed the IP to a domain name, then to allow access from all locations @%
> still nothing,
> 
> Any ideas?

If you look at the firewall logs, do you actually see the connection coming
in to the box?

I'm guessing that the connection may be blocked at the source by some egress
filtering.

If that's not the case, maybe post up some more details about the config, etc.

Have you checked to see if you're mysql is responding to any requests you make
from the same subnet?  Maybe nobody but localhost can connect to the box.

Lastly, the ip address permission system stuff has changed in recent versions
of mysql from a flat ip to ip/netmask notation (e.g. 10.10.0.0/24).

HTH,
