I'm a web developer myself - not much of a graphical designer but I work

online demo is coming any week now...

Drupal is extremely scaleable - which is one of the primary reasons  
we chose it. We are able to run multiple community sites using a  
shared codebase. We also have a few shared tables for things like  
roles and permissions - as well as content types. This means we don't  
have to reinvent the wheel as we add communities. Most of the work is  
achieved at the theme level - core never has to be touched.

Is it me or is the contrib stuff for Mambo really scary? I have never  
used it but this is one of the reasons why we chose drupal over mambo/ 
joomla for fresnofamous.com. Drupal makes it totally clear which  
contrib modules are in full release vs CVS. I found most tools (plug- 
ins? drupal calls them modules) I was looking for were still in beta.

> However, the availability of
> attractive, well-designed templates is very different: I couldn't find
> almost any for Drupal, while there are plenty of useful and tweakable

> Also, most Drupal sites that I have seen
> seem to be kind of hard to navigate - as a user, it's easy to feel  
> lost
> or not realize what else there is to see on the site.

Interesting! I think there will be a lot more themes available for  
the next release. FWIW It's reallyreally easy to code a custom theme  
for Drupal - or to alter an existing one. I usually start with  
bluemarine (phptemplate theme engine is a must) and tweak from there.  
Configuring navigation is the job of the site admin - and I agree it  
is not always done well on Drupal sites (true of many sites in  
general). This is part of the reason I customize themes pretty heavily.

There certainly is a LOT of contrib stuff for Mambo.  Much of it is good.  And
many of them are alpha or beta - I think it is wonderful that people are
upfront about the status. Many of them eventually graduate to "stable" 
quality,
depending on community involvement, feedback and contributions. Others fall by
the wayside, because they are superceded by something better or because 
there's
too little interest. I don't consider something that doesn't seem "active".

There's also a whole cottage industry of for-fee plugins that do some really
fancy stuff.

What's scary about Mambo/Joomla in a way is how easy it is to install plugins
(there are specific terms for different kinds of plugins) - they can be
installed by uploading the zip file directly through the back-end web
interface. So if someone else is a Super Administrator, they might install all
kinds of hokey plug-ins that they might like (bells and whistles!!). Also,
web-based installs mean that the file permissions on the server must be set
permissively, allowing the web server to write to folders where it also
executes code - not a good security practice, at all, at all.
