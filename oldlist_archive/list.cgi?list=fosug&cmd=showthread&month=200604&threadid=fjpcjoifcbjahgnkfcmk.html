> Have you used any AJAX-y functionality by the way?

I think most of the google maps stuff i've done on fresnofamous.com  
is considered AJAX - hooks into the geocoder.us lon/lat lookups. This  
was all done with drupal functionality in mind. For other stuff I  
have used yahoo maps/geocoder API in the same way - using curl to  
work around a particular server environment to read xml data and load  
the maps (via google or yahoo) accordingly. I think the phpflickr  
experiments I've done fit here as well. I've been really happy with  
the user-level enhancements this technology can bring into the mix.

What do you need AJAX for? I'm always happy to share code.

> Opinions on which toolkits
> are easier to work with than others?

There are a number of good AJAX frameworks/APIs/toolkits/libraries  
out there. Many are connected. Some of these libraries may be behind  
the stuff I worked on but i am not 100% sure at the moment. It's been  
a few months (6?) since I did any real work on this stuff + I was  
hacking, not necessarily writing from scratch. If I had time to play  
today here's where I would go (no particular order). I've heard good  
things about each of them.
sajax
http://www.modernmethod.com/sajax/
Xajax
http://www.xajaxproject.org/
Dojo
http://dojotoolkit.org/
Prototype
http://prototype.conio.net/
script.aculo.us
http://script.aculo.us/
Rico
http://openrico.org/

I read that there's a toolkit in the making for Eclipse - don't know  
status.
That's one app I am really interested in testing out.

hth!
