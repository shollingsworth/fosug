> Hi Ronnie,
>
> first, in your startup options, you need to enable PHP it usually looks
> something like this. The following is all one line. Take into account MS
> path names are different than *NIX pathnames.
>
> /usr/sbin/apache2 -D DEFAULT_VHOST -D PHP4 -d /usr/lib/apache2
> -f /etc/apache2/httpd.conf -k start
>
> httpd.conf is going to be your main configuration file. In that file,
> you should have something that looks like this (again, remember *NIX
> paths are different than MS paths)
>
> <Directory "/var/www/localhost/cgi-bin/">
>     AllowOverride None
>     Options None
>     Order allow,deny
>     Allow from all
> </Directory>
>
> That's your cgi directory so if you point your browser to:
>
> http://localhost/cgi-bin/myscript.php it should run it for you. If it
> gives you an error like "permission denied" it's usually a permissions
> issue. Make sure the file is world readable. I'm not sure how it works
> as far as setting the executable bit. I'm thinking it probably works
> differently in windows than it does in *NIX.
>
>
> in httpd.conf you want to make sure that apache is going to see your php
> file as an index file. This is what it looks for if you don't type the
> full path like http://localhost/index.html
>
> DirectoryIndex index.html index.htm index.php
>
>
>
> Give that a try and see how it works for you. You actually got apache
> running so that's not the problem. You're halfway there, you just need
> to adjust some settings, make sure the php module is loaded, and the
> scripts are readable and executable by the world.
>
> Hope this helps.
>
