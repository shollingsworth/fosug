Microsoft has now officially (unofficially) released Internet explorer 7.
This update is being rolled out automatically to people in certain
circumstances.  I don't know what they are yet but they are getting updates
automatically.  It appears that this causes some problems with Novell and
(non-microsoft AD) file servers.  In order to access files from mapped
drives you have to add the Novell file server (non-ms)  IP and UNC. name to
"internet explorer trusted intranet sites" weird yes I know but it is needed
to fix the problems.

You can fix such problems by following these directions:

   1. In Internet Explorer, from the Tools menu, select Internet Options...
.

   2. Under the Security tab, click Local intranet and then click Sites... .

   3. Click Advanced... . In the "Add this Web site to the zone:" box, type
*.domain.com (where domain.com is your company's top-level domain name) or
an IP address range (for example, 157.54.100-200.* ).

   4. Click Add, and then click OK three times to close all the dialog
boxes.


This is the "Cause" According the MS.
