*nod*  There are some difficulties with linux and wifi vendors.  Why this is,

Look to the source young man. [0]  Most of the well supported wireless devices
are listed there.  Once you find a card that meets the specs you want, buy it

there have to say about support.

Wireless vendors are very reluctant to release the specs for their 
radios because they are almost all software controlled  which can 
broadcast at frequencys and power outputs that are not within the FCC 
specs. 

Sadly it's getting worse too especially if you want to use a mini-pci 
internal radio.  HP and IBM (others too I'm sure) are both being forced 
to comply with some FCC policies that require the laptop to only boot 
with a "certified" radio and antenna combination.  If you swap out the 
mini-pci card for a linux/bsd compatible card then the laptop will 
refuse to boot, this check is done by the bios so it's not something 
that is easy to hack around.  I have an HP laptop that I've attempted to 
get a non-certified radio to boot with and it's not been fun.  I've put 
the project way back on the burner and I'll pick it up again later.  To 
sum it up you have to change the vendor ID for the card and then hack 
the drivers.

If you want a working card buy something older that is based on the 
Atheros chipset if you want the best Bsd/Linux support.  The NDIS 
Wrapper project works from what I've heard but you will not be able to 
do any wardriving or any of the fun stuff :(.  Also check every spec on 
the card before you buy it you're going to need to know which hardware 
revisions for sure because sneaky vendors like to change chipsets 
between versions.
