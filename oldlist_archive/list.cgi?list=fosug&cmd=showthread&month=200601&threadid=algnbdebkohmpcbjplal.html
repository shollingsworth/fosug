Minicom rocks.  It's the de-facto standard for linux comms.

As for how I would address your problem, I wouldn't futz with the serial cable
at all.  Connect a machine to that box and nothing else.  Get the mac address
for the switch off the bottom of the box (it should be down there somewhere,
yes?) and then do an arp -s [0] for the mac address of the switch to set the ip to
something on the same subnet as the machine you plugged into it (you'll have
to setup a fixed address on the computer with the default gateway either empty
or set to the ip you set on the computer).

HTH,
--Robert

of your computer is 192.168.1.1/24, 'arp -s 11:22:33:44:55:66 192.168.1.2'.

From: George Fragos ####@####.####

Don't assume that all telecommunications equipment has properly implemented
Internet standards.  Cisco is famous for creating their own "standards."  I
used to build network access equipment and we had to have configuration
parameters that worked with cisco or everyone else's routers.  Many devices
with telnet access only support one session.  My favorite is Motorola.
Their cell site radios have two Ethernet ports for redundancy of management
connections.  Their configuration gives both ports the same IP address.  I
had to build a hub with a failure switch to allow connection of both
management port to the Internet.

George
