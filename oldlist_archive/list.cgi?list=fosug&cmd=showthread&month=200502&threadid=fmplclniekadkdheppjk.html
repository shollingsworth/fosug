Bzzz.  Wrong.  This is _not_ an encryption algorithm.  It is a one way hashing
algorithm (a la MD5) that is used to sign data.  The problem with it being
'broken' is that there exists multiple sets of data that hash to the same
value.  This means theoretically that I could create a document and claim that
stevo wrote it by making it match a previously written document that he'd
signed.

SHA-1 is used to do authentication, but as long as you keep /etc/shadow safe,
you shouldn't have to worry about your passwords either.
