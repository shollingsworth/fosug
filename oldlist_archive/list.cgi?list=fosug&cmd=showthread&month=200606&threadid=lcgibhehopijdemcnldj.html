I haven't connected to the City Free Wifi Network but chances are that
they are running some sort of NAT setup. I take that back I'm POSITIVE
they'd be running a nat setup. Basically what happens is you get a dhcp
lease (ip address) when you logon to the network. It would be something
like the following which are called "private" networks.

10.20.12.1 or 192.168.55.0 or 172.16.29.55

From here they go through a router and come out the other end looking
like they came from a public ip address like 72.14.207.99 (which is an
ip for google.com)

So considering that the network probably has 100's of users connected at
the same time coming from one public ip (see above) and enough people
are going to fresnofamous.com (which very well could be ;-) your
security measures on the website thought the city wifi public IP was
being baaaaaaaaad.

No, but I would be careful what sites and certificates you "accept"
while on the network. Sometimes hackers will act as "rogue" routers and
intercept traffic for the express purpose of stealing passwords, login
information, etc etc. Make sure all your traffic where you send out
usernames and passwords are encrypted. Plain text is plain death on a
public wifi like that. All you'd need is a sniffer and a little know
how.

Not that I've done anything like this :-P

If you're going to connect to a public wifi network like that I'd use a
VPN and some firewall rules that redirect traffic through your encrypted
tunnel. It'll be a little slower, but you can be more confident that the
traffic isn't going to get picked up by somebody in transit.

When you login to the network again, can you post what private IP
address and netmask it gives you?

You can do this by logging into a shell then typing ifconfig

Also visit the site http://whatismyip.com to see what your public
(internet) ip address is.

As far as FresnoFamous.com is concerned, find out the public IP address
is and whitelist it maybe so that that error doesn't occur. But
honestly, I'm not sure I'd disable the feature for the IP because I can
guarantee there are crackers hooked in the network and doing nasty
little cracker things.

Hope this helps :-)
